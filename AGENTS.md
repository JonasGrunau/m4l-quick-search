# m4l-quicksearch

## Purpose
Repository root of **M4L QuickSearch** тАФ a Spotlight-style device & plug-in search overlay packaged as a Max for Live audio-effect device (`.amxd`). Hit a hotkey, a floating search card opens centered over Ableton Live, fuzzy-type a device/plug-in name, press Enter, and it loads onto the currently selected track. This file is the top-level entry point: it holds the npm script table, the TypeScript config, and the map of the four moving parts (the v8 TypeScript brain, the jweb overlay UI, the Node-for-Max bridge, and the Python Remote Script that walks Live's browser). Targets Live 12.2+ with Max 9+ (v8 engine + jweb); macOS is primary.

## For AI Agents

### Working In This Directory
- **Four runtimes, one socket.** v8 (Max) and Max itself cannot open sockets, so `node/bridge.js` relays JSON between the v8 brain and the Python Remote Script over **localhost TCP port 32985**. The browser walk and `load_item` MUST live in the Python script тАФ Live's device *browser* is not exposed to the Max LiveAPI (only tracks/devices are).
- **`dist/` is generated, never hand-edit.** It is git-ignored and regenerated by `npm run build`. Edit `src/`, `html/`, or `tools/` sources and rebuild.
- **Load-time crash risks the build can't catch** (guarded in code, do not regress): the patcher's `project` dict must carry `searchpath`/`layout` or Max segfaults deserialising the `.amxd`; LiveAPI observer callbacks must not read the API inline (Live re-entrancy limit crashes Max тАФ the selection watcher defers reads to a scheduler tick).
- **No runtime npm dependencies.** Only `esbuild` + `typescript` as devDependencies; `node/bridge.js` is Node stdlib (`net`) only. The opt-in `node/global-hotkey.js` pulls `uiohook-napi` via its own `script npm install`, not the root install.
- **The overlay is inlined into the v8 brain** (`tools/bundle-ui.mjs` тЖТ `__QS_UI_HTML__`, baked in via esbuild `define`) and handed to jweb as a self-contained `data:` URL тАФ so only `dist/` needs to be on the Max search path, and a frozen `.amxd` is self-contained too. `html/` is build-time source, not shipped. Releases zip `dist/` + `node/` + `remote-script/` + `INSTALL.txt`.

### Testing / Verifying Changes
- `npm test` (`tools/test.mjs`) тАФ unit tests for search/compat/bridge logic.
- `npm run typecheck` (`tsc --noEmit`) тАФ strict type-check of `src/` + `types/`.
- `npm run build` тАФ esbuilds `src/` тЖТ `dist/quicksearch.js`, generates the patcher JSON, wraps the `.amxd`, and self-checks by parsing the device back through Ableton's `amxd_textconv.py` logic.
- **Automated checks cannot prove the device loads.** Always smoke-test in Live: drop `dist/QuickSearch Dev.amxd` on a track and watch the Max Console for `QuickSearch: init (browser bridge)` then `QuickSearch: indexed N items` with **N > 0** (`0` or a *"Waiting for the QuickSearch Remote Script"* overlay means the Remote Script isn't selected). See the README *Verify-in-Max checklist*. The single highest risk is jweb keyboard capture (the spacebar leaking to Live's transport).

### Common Patterns
- **Two independent dev loops.** UI/design (`html/`) тЖТ `npm run dev` (`tools/dev-server.mjs` serves `html/` at `http://localhost:5173` with live reload; no Max bridge тЖТ renders sample devices in design-preview mode). v8 logic (`src/`) тЖТ `npm run watch` (rebuilds `dist/quicksearch.js` on save; reload in Live by sending the `v8` object a `compile` message тАФ auto-watch is intentionally off to avoid leaking Live API observers).
- **State crosses bridges as base64-JSON in a single Max atom** (no escaping). Device-scoped Max sends `---qs_ui` / `---qs_from_ui`; the floating window is shown/hidden via `pcontrol`.
- **Compatibility is predicted before loading and confirmed after** by the selected track's device-count delta (`load_item` silently no-ops on an incompatible drop).

## Dependencies

### Internal
- The root npm scripts orchestrate everything: `tools/` reads `src/` (+ `types/` for type-checking) and emits `dist/`; the dev server serves `html/`; the device loads `dist/quicksearch.js` (brain тАФ with the overlay UI baked in) off the Max search path, with `node/` alongside.
- `tsconfig.json` scopes the TypeScript compile to `src/` and `types/`.

### External
- **esbuild** тАФ bundles `src/*.ts` тЖТ `dist/quicksearch.js` (single strict-mode IIFE for the Max `v8` object).
- **typescript** тАФ `tsc --noEmit` type-checking only (esbuild does the emit).
- **Node stdlib** тАФ `tools/` (build/test/dev server: `http`, `fs.watch`, SSE) and `node/bridge.js` (`net`) use no npm packages.
- **Max for Live runtimes** тАФ the `v8` JS engine, `jweb` (overlay), and `node.script` (Node for Max). Requires Live 12.2+ / Max 9+.
- **LiveAPI** тАФ exposes tracks/devices (used for compat classification + load confirmation), but NOT the browser.
- **uiohook-napi** тАФ native OS-global key listener, only for the opt-in `node/global-hotkey.js` (unsigned native module; off by default).
